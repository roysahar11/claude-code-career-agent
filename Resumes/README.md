# Resumes

This directory contains your base resume JSON files, generated by the `/setup` skill during initial configuration.

## Resume JSON Schema

Each resume is a JSON file with the following structure:

```json
{
  "name": "Your Name",
  "title": "Your Professional Title",
  "contact": {
    "email": "you@example.com",
    "phone": "+1234567890",
    "linkedin": "linkedin.com/in/yourprofile",
    "github": "github.com/yourusername",
    "location": "City, Country"
  },
  "summary": "A brief professional summary...",
  "technicalSkills": [
    {
      "category": "Category Name",
      "skills": [
        { "name": "Skill Name", "highlight": false }
      ]
    }
  ],
  "experience": [
    {
      "title": "Job Title",
      "company": "Company Name",
      "location": "City, Country",
      "dates": "2024 - Present",
      "bullets": [
        "Achievement or responsibility description"
      ]
    }
  ],
  "education": [
    {
      "degree": "Degree Name",
      "institution": "University Name",
      "dates": "2020 - 2024",
      "details": "Optional details"
    }
  ],
  "certifications": ["Certification Name"],
  "languages": [
    { "language": "English", "level": "Native" }
  ],
  "hobbies": ["Hobby 1", "Hobby 2"]
}
```

## File Naming

- `base-devops.json` — Base resume emphasizing DevOps/infrastructure skills
- `base-fullstack.json` — Base resume emphasizing full-stack development skills
- Add more base variants as needed for different target roles

## How Resumes Are Used

1. **`/customize-resume`** — Takes a base resume and tailors it for a specific job posting
2. **`/quick-apply`** — Automatically selects the best base resume for each job and creates a draft
3. **`/scan-jobs`** — Scores each base resume against job postings to recommend the best starting point

Customized resumes are saved in `Applications/{company}/{role}/drafts/` and finalized versions in `Applications/{company}/{role}/final/`.
